# Getting Started

The Flaunch protocol offers fantastic benefits for both creator revenue and community support. However, the key requirement for this functionality is that the token is deployed through the Flaunch platform. This is done so that we can ensure the security of the token and facilitate our fair launch tech.

But not all tokens can came from such Flaunchy beginnings. If you have an existing token from another protocol already deployed, or you would like to build a protocol on top of Flaunch, then we have built the `AnyPositionManager` for you.

The `AnyPositionManager` allows for **approved**, external ERC20 tokens to benefit from the bespoke Flaunch hooks.&#x20;

### Differences of a non-native Flaunch token

* The fair launch element will be removed, as the token will have already been through price discovery externally.
* The "creator" of the token will not receive revenue as a Flaunch token would. Instead LPs will be rewarded, promoting better liquidity for migrated tokens. This will mean that the ERC721 ownership won't hold benefit, but will act as a badge of honour for the first person to create the position.
